# ポイントシステム仕様

## 概要
ユーザーのアクション（ゴミ拾い、イベント参加、招待、プロフィール入力など）に対して付与されるポイント（ヒコポ）の仕様。

## 獲得アクション一覧

| アクション | 獲得ポイント | 条件・制限 | 備考 |
| --- | --- | --- | --- |
| ゴミ拾い（ランニング） | 距離/時間に応じた変動 | GPS連動 | 基本機能 |
| 招待コード利用（招待された側） | 500pt | 初回登録時のみ | |
| 招待コード利用（招待した側） | 500pt | 無制限 | |
| 学生相互認証（認証した側） | 100pt | 同一学校の未認証学生に対して | 最大3人まで（認証完了まで） |
| プロフィール入力完了 | 200pt | 全必須項目を入力した時点 | 初回のみ |
| デイリーログイン | 10pt | 1日1回（JST 0:00切替） | アプリ起動時に付与 |
| 誕生日ボーナス | 500pt | 誕生日の初回ログイン時 | 年1回 |

## プロフィール入力完了ボーナス
### 獲得条件
以下のプロフィール項目を全て入力・保存すること。
- お名前
- 性別
- 生年月日
- 居住地（都道府県、市区町村、地域、詳細エリア）
- 興味関心タグ（1つ以上）
- 学生の場合：
  - 学校名
  - 学年
  - 学部（大学生の場合）
  - 最寄り駅

### 実装詳細
- プロフィール更新時（`updateProfile` Server Action）に全項目の入力状況をチェック。
- 初めて全項目が埋まった場合、`is_profile_completed` フラグを TRUE にし、`points` に 200 加算。
- クライアント側で完了アニメーションを表示。
